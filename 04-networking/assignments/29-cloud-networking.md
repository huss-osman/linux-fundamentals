# Cloud Networking

## Overview

This section introduces **cloud networking** and how networking concepts are applied within **cloud environments**. It focuses on core components such as **VPCs, subnets**, and **gateways**, which allow cloud resources to communicate securely and efficiently.

Understanding cloud networking is essential for working with **AWS, cloud infrastructure, containerized applications**, and modern **DevOps environments**.

---

## What You’ll Learn

This section is structured to help you understand the foundations of networking in the cloud:

- **Introduction to Cloud Networking**  
  Understanding networking concepts in **cloud environments**

- **VPCs (Virtual Private Clouds)**  
  Learning how isolated **cloud networks** are created and managed  

- **Subnets**  
  Dividing **cloud networks** into smaller segments for organisation and security  

- **Gateways**  
  Understanding how **cloud resources** communicate with external networks  

- **Routing in the Cloud**  
  Exploring route tables and traffic flow inside **cloud environments**  

- **Security Groups and NACLs**  
  Controlling inbound and outbound traffic for **cloud resources**

- **Container Networking**  
  Understanding networking in **Docker and Kubernetes** environments
    
---

## Commands

These tools and commands help inspect and troubleshoot cloud networking:

- `ip addr` → View network interfaces and IP addresses
- `ip route` → View routing tables
- `ping` → Test connectivity between systems
- `traceroute` → Track network paths
- `docker network ls` → View Docker networks
- `kubectl get svc` → View Kubernetes services

---

## Key Takeaways

- **Cloud networking** applies traditional networking concepts in cloud environments
- **VPCs** provide isolated private cloud networks
- **Subnets** improve organisation and segmentation
- **Gateways** allow communication with external networks
- **Security Groups** and **NACLs** help secure cloud infrastructure
- Understanding cloud networking is essential for **DevOps and cloud engineering**
