# Git Common Commands

## Overview

**Git commands** are the primary way developers interact with a **Git repository**. They allow users to track changes, record project history, and manage collaboration across teams.

By using **Git commands**, developers can stage changes, create **commits**, inspect **repository history**, and synchronise work with remote repositories. Understanding these core commands forms the foundation of working effectively with **Git** in modern **DevOps workflows**.

---

## Commands

```bash
git init      # initialise a new Git repository
git add       # stage changes for the next commit
git commit    # record a snapshot of staged changes
git status    # show staged and unstaged changes
git log       # display commit history
git diff      # show changes between commits or working files

git config    # configure user name, email, and settings
git help      # display documentation for Git commands
git clone     # copy a remote repository
git rm        # remove files from the repository
git mv        # rename or move tracked files
git restore   # undo file changes
```

---

## Key Concept

**git init**  
Initialises a new **Git repository** by creating the **.git directory**, which stores the repository’s **history** and **configuration**.

**git add**  
Moves file changes into the **staging area**, preparing them for the next **commit**.

**git commit**  
Creates a **snapshot** of staged changes and adds it to the **repository history**.

**git status**  
Displays the current state of the **working directory** and **staging area**.

**git log**  
Shows the **commit history**, including commit messages, authors, and timestamps.

**git diff**  
Displays the differences between file versions or **commits**.

**git clone**  
Creates a local copy of a **remote repository**.

**git config**  
Used to configure **Git settings**, such as username and email.

---

## Key Takeaways

- **Git commands** allow developers to interact with a **Git repository**

- **git add** stages changes before committing them

- **git commit** records a **snapshot** of the project

- **git status** and **git diff** help inspect **repository changes**

- **git clone** creates a local copy of a **remote repository**

---

## Reflection

Learning the most common **Git commands** is essential for working with **version control systems** effectively. These commands provide the tools needed to **track changes**, manage **project history**, and collaborate safely with other developers using **Git repositories**.
