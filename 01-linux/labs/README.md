This folder contains **hands-on Linux practice labs** completed as part of my DevOps training.

The focus is on applying Linux fundamentals in realistic scenarios and building practical troubleshooting skills.

Labs emphasise investigation, command-line reasoning, and understanding how Linux behaves in real environments.

---

## Bandit (Levels 1–12)

The **OverTheWire Bandit** wargame is used to reinforce Linux fundamentals through progressively harder challenges.

Each level focuses on understanding core command-line tools and Linux behaviour.

These challenges strengthen skills in:

- File discovery and navigation
- Searching file contents
- Permissions and ownership
- Encoding and decoding data
- Working with hidden files and unusual filenames

Each level documents:

- The **challenge objective**
- The **commands used**
- The **reasoning behind the solution**
- The **key concept being learned**

Bandit is treated as a **learning exercise**, not a puzzle to brute-force.  
The emphasis is on understanding Linux behaviour rather than memorising solutions.

## SadServers (Easy, Medium, Hard)

**SadServers** scenarios simulate real-world Linux failures that engineers might encounter in production environments.

Each scenario requires investigating a broken system and identifying the root cause.

Common troubleshooting areas include:

- Services failing to start
- Misconfigured permissions
- Broken networking configurations
- Disk or filesystem problems
- System process issues

Notes focus on:

- Investigating symptoms methodically
- Using Linux tools to diagnose issues
- Understanding the underlying system behaviour
- Explaining why the fix works

These exercises help develop **production-grade troubleshooting and operational thinking**.
