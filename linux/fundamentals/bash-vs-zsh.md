# Bash vs Zsh

## Overview

**Bash** and **Zsh** are popular Unix shells used to interact with the operating system through a command-line interface. They allow users to execute commands, run scripts, manage files, and automate tasks efficiently.

While both shells provide similar core functionality, they differ in features, customisation options, and user experience. Understanding how shells work — and the role they play in interpreting commands — is essential for working effectively in Linux environments and building strong scripting foundations.

---

## Bash
- Default shell on most Linux systems
- Widely supported and script-friendly
- Common for servers and automation

---

## Zsh
- More advanced auto-completion
- Better theming and plugins
- Popular for local development environments

> [!NOTE]
> **Key Note:** Shell scripts are usually written for Bash to ensure compatibility across systems.

---

## Reflection
Understanding the differences between **Bash** and **Zsh** helped me focus on Bash as the primary shell for scripting and automation, since it is widely used in production environments and DevOps workflows. Its compatibility and predictable behaviour make it the safer choice when writing scripts intended to run across different systems.

At the same time, I began to appreciate Zsh for its enhanced usability features, such as improved tab completion and customization options, which make daily terminal use more efficient and enjoyable. This distinction clarified when to prioritise portability and scripting reliability versus personal productivity and convenience.
